分享一个我用 AI 辅助开发的小工具 —— nasdash，一个跑在飞牛 OS 上的硬件监控面板

一、应用简介

  • 用 AI 辅助开发：从界面到各项检测逻辑，基本是和 AI 反复搓出来的，算是 AI 练手 + 自用两不误的产物。
  • 干什么用的：把飞牛 NAS 的硬件状态集中到一块面板里看 —— 阵列卡、硬盘 SMART、温度、风扇、存储卷、网卡、GPU 全都有。
  • 形态：飞牛 OS 第三方 fpk 应用，装完在桌面点图标直接在窗口里打开（不跳浏览器）。基于 Python + Flask 实现，依赖少，打包成 fpk 后装完即用。
  • 当前版本：v1.6.8（开源，GitHub 见文末）。

二、主要功能

  • 阵列卡监控（MegaRAID / LSI）：控制器温度、CacheVault 状态、每块硬盘的型号 / 接口 / 状态。
  • 硬盘健康（SMART）：温度、健康状态、SAS / SATA 识别、型号容量，坏盘一眼可见。
  • 系统资源：CPU 型号 / 占用 / 温度、内存、主板传感器、PCH 芯片组温度、网卡、GPU。
  • 存储卷（mdadm）：RAID 级别、成员盘、降级预警（如 [ U_ ] 状态直接标红）。
  • 风扇控制：实时转速、当前 / 目标占空比，支持手动调速与自动 / 手动模式切换。
  • 检测新版本：面板自动比对 GitHub 最新 Release，发现更新顶部弹横幅 + 下载链接；也有手动「检查更新」按钮。

三、实际运行截图

下面是我自己这台机器上的实测数据，均由 nasdash 真实抓取。

1. 总览面板（硬件配置检测）

图中：Pentium Gold G5400 / 43°C / 7.6 GB 内存；顶部状态栏实时显示散热风扇、阵列卡、SAS 硬盘、SATA 硬盘、Docker 的运行 / 健康状态。

2. 阵列卡详情（LSI MegaRAID）

图中：LSI MegaRAID SAS 9271-8i，控制器芯片温度 62°C，CacheVault 40°C，物理盘 14T SAS 以 JBOD 模式列出。阵列卡类型、固件版本、驱动版本都直接读出来。

3. 硬盘 SMART（坏盘红色告警）

图中：2 块希捷 14T SAS 硬盘健康 OK（51°C），另有 1 块三星 PM871b SSD 已显示为 FAILED。这就是 nasdash 最想帮大家「一眼发现」的问题 —— 坏盘不再藏在命令行里。

4. 系统资源（温度 / 风扇 / 电压）

图中：CPU 1 分钟负载 1.84，内存占用 31.9%；CPU Package 43°C、PCH 芯片组 72°C；3 个 PWM 风扇转速（其中 1 个 0 RPM 停转）、主板各电压全部列出。

5. 风扇控制

图中：三把 it8772 风扇，支持一键全速 / 10%~90% 预设，以及单风扇手动 PWM 调节。默认是曲线温控，也可以切手动模式。

6. 存储卷（mdadm / 容量）

图中：RAID0（12.7 TB）与 RAID1（698.5 GB / 111.7 GB）阵列状态；各挂载点（/、/fs、/vol1、/vol2）的总容量、已用、可用、使用率进度条，一眼看满。
