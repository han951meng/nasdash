# nasdash 操作手册（大白话版）

本手册讲清楚 nasdash 这面板**怎么用、每个开关干什么、遇到问题怎么查**。不用懂 Linux 命令，照着点就行。

> 本手册**仅对应当前代码版本 v1.8.5**。以后 nasdash 升级出新版本，本手册会同步更新到对应版本号，老版本用户请回 GitHub Releases 找对应 tag 的手册查看。功能以你实际界面为准，个别主板/风扇命名可能不同，属正常现象。

**本版本修复了什么**

==**① 硬盘温控：SAS 企业盘 / 固态盘 不再误判为机械盘**==
==  v1.8.0 起把 `sde` 这类 SATA 固态（内核 `rotational=0`）当普通机械盘处理，盘永远不睡就把「全部休眠 → 停风扇」卡死，硬盘休眠停转功能一直失效。本版改为自动识别为「常驻」并排除在停转判断外，机械盘睡了风扇就能停；同时把 SAS 企业盘也排除在停转判断外（厂商锁死不让停）。==

==**② 温控配置「存得进、读不出」修复**==
==  早期版本有个隐蔽 bug——温控配置（硬盘温控的勾选、曲线、受控风扇列表）能写盘但读不出来，导致每次刷新或重启就回到默认值、看起来「勾又没了」。现已修复，保存后刷新/重启配置都还在。==

---

## 一、先搞明白：nasdash 是什么

nasdash 是飞牛OS（fnOS）上的一个**硬件监控面板**，把原本要 SSH 敲命令才能看的硬件状态，做成网页一眼看完：

- 你有什么硬件（CPU/内存/主板/显卡/阵列卡/网卡）
- 硬盘健不健康、多少度
- 风扇转多快、能不能调速
- 阵列卡/Raid 状态、存储卷用了多少
- Docker 容器在不在跑
- 温度高了会不会自动报警

一句话：**它是你 NAS 的「仪表盘 + 风扇遥控 + 报警器」三合一**。

---

## 二、怎么打开

1. 在飞牛OS **应用中心**装好 nasdash（手动安装 `nasdash.fpk`）。
2. 桌面会出现「NAS硬件监控」图标，点开即进入。
3. 面板经飞牛**统一网关**暴露，**要先登录飞牛账号**才能看。
4. 外部访问请走飞牛自带的远程访问 / 反代，**不要直接把 9800 端口暴露到公网**。

> 小提示：面板每 30 秒左右自动刷新（系统资源页有「自动刷新」开关）。

## 三、七个模块逐一讲解

左侧边栏共七个模块，下面挨个说。

### 1. 硬件配置检测（首页仪表盘）

进面板第一眼就是它，单页总览：

- **顶部状态栏**：阵列卡 / 硬盘 / Docker / 系统负载一屏速览。
- **系统配置**：
  - 处理器（型号、核心/线程数）
  - 显卡（有独显才显示）
  - 内存（品牌取自 SPD 直读，会区分「模组厂」和「颗粒厂」——比如模组是某品牌、颗粒是三星）
  - 主板型号（DMI 读不到时，可网页手动标注，比如「豆希 WB360」）
  - 系统信息（含已运行多久）
  - 网络（网卡 IP / 速率 / 状态）
- **阵列卡 & 磁盘**：卡型号 + 物理盘，SMART 健康速览。

> 没看懂的字段不用管，这一页就是「看个全貌」。

### 2. 阵列卡

面板用 `lspci` 自动识别你插的是哪种卡，分三种情况显示，**不会瞎报**：

- **MegaRAID 阵列卡**（如 LSI 9271-8i，RAID/IR 模式）：显示型号 / 序列号 / SAS 地址 / 固件 / BIOS / 驱动 / PCI 地址、CacheVault 缓存电池状态、**ROC 芯片温度**、JBOD 物理盘列表（槽位 / 品牌 / 型号 / 接口 / 容量 / 状态 / 转速）。
- **HBA 直通卡**（如 LSI 9300，IT 模式）：显示卡型号 + 「IT 直通模式」说明，额外显示 **HBA 芯片温度**。直通卡下的盘由系统直接识别成 `/dev/sdX`，温度去「硬盘 SMART」标签页看。
- **纯 SATA 主板**（没独立阵列卡）：提示「无独立阵列卡」，温度请见「硬盘 SMART」。

> **双磁臂（双执行器）硬盘**：阵列卡会把每个执行器当独立盘暴露（比如各 7T），面板在阵列卡页显示整盘标称容量（如 14T）并标注每执行器容量——这是正常的，不是 bug。

### 3. 硬盘 SMART

所有硬盘的健康体检页：

- 同时支持 **SAS/SCSI 盘** 和 **SATA/ATA 盘**。
- 自动识别品牌与特性（希捷/西数/东芝/三星，双磁臂盘带徽章）。
- 健康状态：OK/PASSED 标绿，异常标红。
- 温度 / 通电时长 / 坏块计数 / 错误计数。
- **机械盘显示真实转速**（7200 rpm / 5400 rpm）；**SSD 显示「固态(SSD)」**。
- SATA 盘额外显示 Reallocated / Pending / Uncorrectable / UDMA CRC 等告警计数。

> 这块盘到底是不是 SSD，面板靠内核 `rotational` 值判断（0=固态 / 1=机械），准的。

### 4. 系统资源

- **CPU**：型号 / 核心 / 线程 / 频率 / 负载。
- **内存 / Swap**：使用率。
- **温度传感器**：分类显示（CPU 各核心 / 芯片组 PCH / 主板 / ACPI），CPU 温度带 max/crit 上限。
- **风扇转速**：读转速 + PWM 占空比。**0 RPM 的停转风扇会标红**。控制模式从 sysfs 读（0=关闭 / 1=手动 / 2=自动温控）。
- **电压**：+3.3V / 待机 / CMOS 电池等。

### 5. 历史趋势

- 磁盘 I/O（读/写速率）折线图，可切 **24 小时 / 7 天 / 30 天**。
- 数据存在本地 SQLite，**重启不丢**，自动保留 30 天。

### 6. 风扇控制（重点模块）

这是最能「动手」的模块。三块内容：**预设调速 / 两套温控曲线 / 安全接管**。

#### 6.1 预设档位与自定义

- 预设档位（默认 / 全速 / 10%…90%）一键调速，卡片实时显示 RPM 与占空比。
- 每个风扇可**网页改名**（如「CPU 风扇」「机箱风扇」）和标供电电压（12V / 5V / 未知），按安装实例持久化。
- 手动调速下限放宽到 **10%**；点「恢复自动」即把风扇交还主板/内核自动控温。

#### 6.2 两套独立温控曲线

面板能按温度自动调风扇，且**两套互不干扰、可分别接管不同风扇**：

**A. 硬盘温控（disk_temp）—— 看盘的温度调风扇**

适用场景：你想让「硬盘散热风扇」跟着硬盘温度走（盘热就多吹，盘凉就少吹）。

配置步骤：
1. 勾 **「启用硬盘温控」**。
2. **监控硬盘**：勾你想盯的盘（界面每块盘右侧会实时显示状态，见下方「常驻」说明）。
3. **受控风扇**：勾由硬盘温度接管的风扇（比如机箱风扇 / SYS_FAN2）。**这里至少勾一个**，否则等于没接管，面板会弹橙色提示。
4. **温控曲线**：设 4 个值——
   - 开转温度（盘到这温度开始转，如 40°C）
   - 全速温度（到这温度拉满，如 60°C）
   - 开转占空比（最低转速百分比，如 30%）
   - 全速占空比（如 100%）
   - 恢复自动温度（降到这以下就把风扇交还系统自动控温，如 35°C——带滞回，避免临界抖动）
5. 可选「曲线编辑器」自定义「温度→PWM」多点曲线（可一键套用静音/均衡/性能预设）。
6. 点 **「保存配置」**，盯一下按钮旁提示是否出现「已保存 ✓」。

**关于「常驻」—— 这是这次修复的重点**

每块监控盘右侧会显示三种状态之一：

| 显示 | 含义 |
|------|------|
| **温度（如 44°C）** | 这块盘是普通机械盘，能正常休眠，正显示实时温度 |
| **休眠** | 盘已经停转省电了 |
| **常驻** | 这块盘是 **SAS 企业盘**或**固态盘(SSD)**，厂商为数据安全**锁死了不让停转**，所以它永远不参与「全部休眠→停风扇」的判断 |

**那到底要不要勾 SAS/固态盘？** 两种选法，停转功能都正常：

- **勾上（推荐，散热更周全）**：这块盘的温度会**参与风扇曲线控速**——SAS/固态一发烧，受控风扇就提速给它散热。尤其 SAS 企业盘本身容易跑暖，勾上更稳。
- **不勾**：它完全不监控、不查温度、也不影响停转功能（反正天生被排除）。代价是它发烫时风扇不会为它专门提速，得靠机箱整体风道。

> 一句话：**勾＝让它的热量去管风扇；不勾＝当它不存在。** 停转功能两种都不受影响。

为什么要这么设计？因为「硬盘全部休眠才停风扇」这条规则，一旦混了 SAS/SSD 盘就**永远等不到它们睡**（它们天生不睡），等于把整个停风扇功能卡死。所以面板把 SAS/SSD 直接排除在「判断是否该停转」之外：

- **只有能睡的机械盘全都睡了**，风扇才停 → 省电。
- 但安全阀还在：如果某块 SAS/SSD 盘**发烫超过开转温度**，风扇照样转给它散热，不会不管它。
- 界面上 SAS/SSD 盘标「常驻」，让你一眼知道它不参与休眠判断。

> 实战例子：你这台机器 `sde` 其实是一块 SATA 固态盘（rotational=0），之前一直被当机械盘、悄悄堵着停转功能；现在正确识别成「常驻」，机械盘（sdc/sdd）睡了风扇就能停。

**B. 主板/CPU 温控（sys_temp）—— 看 CPU/主板温度调风扇**

跟硬盘温控对称配置，只是温度来源换成 CPU 封装温度或主板温度。两套可以各管各的风扇，互不打扰。

#### 6.3 与 FanControlServer 的共存（安全接管）

飞牛系统自带一个风扇服务叫 **FanControlServer（FCS）**。两套调速器同时写同一个 PWM 通道会抢控制权、转速乱跳。nasdash 的做法是**安全接管**：

- 当 nasdash 正在主动控速（手动/温控/规则接管了某风扇）时，自动先停掉 FCS；
- 当这些风扇**全部交还系统自动控温**后，自动把 FCS 拉起来。
- 你**没开任何控速**时，nasdash 完全不碰 FCS，零打扰。

面板底部还有**一键永久禁用/恢复 FCS** 的开关（显示当前状态）：
- 永久禁用 = `stop` + `disable`（重启不复活）；
- 恢复 = `enable` + `start`。
- 都不用 SSH 敲命令。

> 纠结要不要杀 FCS？**不用手动杀**。只用一个就够：要么用 nasdash 管，要么在面板一键永久禁用 FCS 让系统原生控温，二选一。

### 7. 存储卷

- mdadm RAID 阵列状态（级别 / 成员盘 / 容量）。
- 挂载点容量（总大小 / 已用 / 可用 / 使用率 / 文件系统）。
- 自动过滤 docker overlay / tmpfs 这些非存储卷，不凑数。

### 8. Docker

- 容器状态概览（运行中 / 总数）。
- 容器列表：状态 / 名称·镜像 / 内存占用 / CPU 占用率 / 网络 RX·TX 速率 / 端口映射（自动探测 host 与 bridge 模式）/ 运行时长。
- 已停止的容器标注「容器停止不检测端口」。

### 9. 控制与自动化（告警 + 通知 + 手册）

- **告警规则**：CPU / 主板·芯片组 / 硬盘温度超阈值，或硬盘 SMART 健康异常（pre-fail / 异常）时自动触发。
- **通知渠道**：面板内提示（始终记到应用通知日志）+ 可选 **Telegram / Bark / 邮件(SMTP)** 推送。
- **一键健康报告**：导出全部硬件状态与活动告警为 **JSON**（留档）或 **HTML**（排版报告）。
- **📘 操作手册**：左侧导航「其他」分组下（🔔 控制与自动化下方）有「📘 操作手册」一项，点它会在右侧面板里**就地展开**这份文档（不跳新页面、不离开应用），**离线可读、不用联网**。里面就是你现在看的这份大白话教程。

## 四、配置存在哪？重启会不会丢？

所有设置（风扇名称、温控曲线、硬盘温控开关、受控风扇列表等）都持久化在：

```
/vol1/@appdata/com.dashboard.nasdash/
```

这是飞牛的系统数据盘，**重启不丢**。各配置文件：

| 文件 | 内容 |
|------|------|
| `fan_mode.json` | 风扇手动/自动档位 |
| `fan_disk_temp.json` | 硬盘温控配置 |
| `fan_sys_temp.json` | 主板/CPU 温控配置 |
| `fan_labels.json` | 风扇自定义名称/隐藏 |
| `fan_curve_*.json` | 自定义曲线 |
| `alert_*.json` | 告警规则与通知渠道 |

> **重要修复（本版已修）**：早期版本有个隐蔽 bug——温控配置「存得进、读不出」，导致勾选在刷新/重启后丢失、温控实际不生效。现已修复，**保存后刷新/重启配置都还在**。如果你之前遇到「勾好的设置没了」，升级这版后就没事了。

## 五、常见问题排查

| 现象 | 是不是 bug | 怎么办 |
|------|-----------|--------|
| 风扇显示「无转速信号」 | 不是 | 该通道风扇没接测速线（3针/缺 tach）或空接口。仍能调速，点「隐藏」收起即可 |
| hwmon 编号跨重启变化 | 不是 | 内核给控制器分配的 `hwmonN` 会漂。面板已按通道序号(idx)命中真实通道，不影响调速，重启不乱 |
| 名字显示「it8772 风扇 N」 | 不是 | 旧 FanControlServer 配置残留标签。风扇卡片里改名即可 |
| 手动调速后停目标值不回弹 | 不是 | 设计行为：拖动即手动控制。点「恢复自动」才交还系统 |
| 滑块最低只到 10% | 不是 | 手动下限 10%（该硬件 10% 仍可转），非 0% |
| 勾好的温控设置重启没了 | **曾是 bug，已修** | 升级到含修复的版本即可；若仍丢，检查 `/vol1/@appdata/.../fan_disk_temp.json` 是否生成 |
| 启用硬盘温控但风扇不动 | 多半是没勾「受控风扇」 | 去「受控风扇」勾至少一个风扇再保存；面板会橙色提示 |
| 混了 SAS/SSD 盘，休眠停转不生效 | 已修复 | 升级后 SAS/SSD 标「常驻」自动排除，机械盘全睡即可停转 |

## 六、版本自检

面板会做版本自检（检测到有新版本时在界面提示）。保持更新即可获得 bug 修复与新功能。

## 七、一句话总结操作顺序

1. 装好 → 桌面图标打开。
2. 想调速：风扇控制页 → 选档位或启温控 → 勾受控风扇 → 保存。
3. 想报警：控制与自动化 → 设阈值 + 填通知渠道。
4. 想看历史：历史趋势页切时间范围。
5. 改完设置**点保存**，刷新/重启都在——配置持久化在系统盘。

还有不会用的地方，去 GitHub Issues 提一句就行。
